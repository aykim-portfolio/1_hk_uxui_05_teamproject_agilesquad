1. Continue (좋았던 점 / 유지할 점)
자율적인 역할 분담과 몰입: 랜덤이나 강제 배분이 아닌, 각자 관심 있는 영역(코스피 지수 연동, 디자인 MD, 데이터 분석 등)을 맡아 주도적으로 진행하여 흥미와 재미를 느낄 수 있었음.
자유로운 상호 질의 및 피드백: 작업 중 막히는 부분(예: 코스피 API 지수 활용 등)이 있을 때 서슴없이 질문하고 서로 코드를 공유하며 해결책을 찾아간 과정이 긍정적이었음.
결론 페이지의 활용성: AI(Claude)를 통해 도출된 구글 슬라이드 마지막의 종합 결론 페이지가 전체 전략을 보여주는 데 유용했음. 이를 기반으로 향후 더 발전된 결론 도출이 가능할 것으로 보임.

2. Stop (아쉬웠던 점 / 문제점)
초반 싱크 및 중복 작업 제어 부족: 각자 작업을 시작하기 전 선별 기준이나 범위를 명확히 논의하지 않아, 초반부 작업에서 싱크가 맞지 않거나 중복될 가능성이 컸음.
최종 파일 통합 과정의 혼란: Claude 대시보드 작업물을 구글 슬라이드로 변환하는 과정이 매끄럽지 않아 이를 수정하고 고치는 데 예상보다 많은 시간이 소요됨.
팀원 간의 실시간 내용 공유 부족: 개별 스크린을 띄워놓고 피드백을 받다 보니 '개인 피드백 시간'처럼 인식됨. 본인 수정 작업에 집중하느라 다른 팀원의 분석 내용(특히 트래픽 분석 등)을 완벽히 이해하지 못한 채 넘어가는 경우가 발생함.
디자인 MD(공통 스타일) 적용의 일관성 부족: 동일한 디자인 MD 가이드를 적용했음에도, 3명이 각자 Claude를 통해 반영하다 보니 최종 결과물에서 색상이나 스타일이 완벽하게 통일되지 않고 약간씩 다르게 출력됨.
GitHub 연동 이슈: GitHub와 연동은 시도했으나 플러그인 설치 오류, 파일 위치 혼선, 포트폴리오 연동 여부 불확실 등 팀원마다 연결 상태가 제각각이고 제대로 활용되지 못함.

3. Start (앞으로 시도할 개선 대책 / 액션 아이템)
📊 프로세스 및 협업 방식 개선
산출물 형태 선(先)지정: 프로젝트 시작 시 최종 공유할 파일 형태(예: 구글 슬라이드, 웹 등)를 먼저 확정하고 그에 맞춰 업무 순서를 기획함.
중간 공유 세션 및 질의응답 시간 확보: 개인 피드백 직후 약 10분간 팀원 간 묻고 답하는 리뷰 시간을 공식적으로 가져 내용의 정합성을 맞추고 순서를 조율함.
공통 협업 툴(Figma/FigJam) 적극 활용: 다운로드로 인한 용량 낭비를 줄이고 실시간 싱크를 맞추기 위해, 하나의 피그잼(FigJam) 파일에 모든 자료, 질문, 피드백을 때려 넣고 한눈에 확인할 수 있는 환경 구축.
🎨 디자인 시스템(MD) 보완
색상 가이드 구체화: 한국경제의 상징색(남색) 외에 데이터 지표 구분을 위해 세컨더리/테리토리 컬러 체계를 명확히 정의 (예: 한국경제 프리미엄 라인의 주조색인 레드 계열이나 그레이 톤을 활용하여 일관된 톤앤매너 유지).
UI 컴포넌트 구조화: 대시보드 카드의 배경을 꽉 채우기보다는 회색 테두리(외곽선) 중심의 깔끔한 스타일로 정리하여 시각적 피로도를 줄이고 인포메이션 하이라이트 효과를 높임.
🛠️ 기술 및 인프라 해결
