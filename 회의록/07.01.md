# **회의록: 뉴스앱 프로젝트 스프린트 회의**

## **📋 주요 논의 및 결정 사항**

### **1\. 프로젝트 범위 및 우선순위 재조정**

* **현황/배경**

  * 멘토링 피드백에서 "너무 범위가 넓으니 집중하라"는 의견 수렴  
  * 기사뷰와 유료화 두 가지 주요 기능 개발 중  
  * 지속적으로 "기사뷰에 집중하라"는 의견이 제시됨  
* **주요 내용**

  * 기사뷰와 유료화 중 우선순위 재검토 필요  
  * 기사뷰를 먼저 완성도 있게 마무리한 후 유료화 진행 제안  
  * 두 기능 간 작업 분담의 효율성 문제 논의  
* **결정 사항**

  * **기사뷰(기사 읽기 \+ 쉬운 모드)를 우선 집중 개발하기로 합의**  
  * **유료화 기능은 기사뷰 개발 완료 후 진행하도록 우선순위 조정**  
  * AI 기능(쉬운 모드)을 기사뷰의 일부로 포함하여 함께 개발

### **2\. 디자인 초안 및 명세서 업데이트**

* **현황/배경**

  * 사용자 정리 문서 제작 중(약 50% 완료)  
  * 발표 슬라이드 겸용으로 제작 중이며 어제 변경사항 반영 필요  
  * 필수 아이콘과 네비게이션 바 정의 필요  
  * MD 파일 컬러 정보 업데이트 필요  
* **주요 내용**

  * 신규 화면 상세 디자인 완료 상태 확인 필요  
  * 컴포넌트 제외 범위 재확인  
  * 색상 변경 사항(컬러)이 반영되지 않음  
* **결정 사항**

  * 필수 아이콘과 네비게이션 바 정의를 **오전 중 완료**  
  * MD 파일 컬러 정보 업데이트를 **오전 중 완료** (약 2시간 소요 예상)

### **3\. 프로토타입 개발 전략 및 역할 분담**

* **현황/배경**

  * 기사뷰는 좌측(본문) 영역과 우측(패널) 영역으로 구성  
  * 프로토타입 신속 제작 필요 (내일 오전 발표 예정)  
  * A/B 테스트(ABC 버전 비교) 고려 중  
  * 단일 통합 산출물로 발표해야 함  
* **주요 내용**

  * **2명 분담 vs 1명 개별 수행 논의**  
    * 2명 분담: 진도 빠름, 하지만 통합 어려움, 크레딧 많이 소비  
    * 1명 수행: 통합 용이, 진도 느림, 크레딧 효율적  
  * 프로토타입 개발 사이클 직접 테스트 필요 (실제 소요 시간/비용 파악)  
  * 기사뷰 우측 패널 영역을 우선 테스트 운영으로 결정  
* **결정 사항**

  * **역할 분담 결정 전 프로토타입 테스트 실행으로 실효성 검증**  
    * 민성님: 오전 업무 완료 후 약 5분 정도 프로토타입 테스트 운영  
    * 간단한 버전으로 합치기 가능 여부 판단  
  * **프로토타입 개발 담당: 주연님(패널/우측 영역) \+ 다빈님(기사뷰/좌측 영역 쉬운 모드)**  
  * 패널 영역은 기존 작업 방식 유지 (화면별 분담)  
  * **여러 버전 산출 시 버전 관리**: 명확히 더 발전된 버전만 진행, 헷갈리는 경우 팀 협의 후 선택

### **4\. 발표 준비 및 자료 취합 프로세스**

* **현황/배경**

  * 내일(7월 3일) 오전 발표 일정 확정  
  * 발표자 미선정, 자료 취합 방식 미정  
  * 발표 준비 시간(약 1시간) 필요  
* **주요 내용**

  * 발표 담당자 선정 필요  
  * 자료 취합 방식: 1명이 초안 작성 → 팀 피드백 → 수정의 효율성 제시  
  * 최종 합의 및 검수 시간 필요  
* **결정 사항**

  * **오늘 오전 중**: 프로토타입 반복 개발, 발표 자료 제작  
  * **오늘 오후 4시까지**: 발표자료 최종본과 프로토타입 완성 목표  
  * **오후 4시\~5시**: 팀 전체 최종 검수 및 합의 (필요시 오후 5시까지 연장)  
  * **발표 자료 취합**: 1명이 통합 초안 작성 후 팀 피드백 수렴하는 방식으로 진행

### **5\. 사용자 테스트/AB테스트 일정 조정**

* **현황/배경**

  * 원래 계획: 프로토타입 기반 사용자 테스트(AB테스트) 진행  
  * 프로토타입 개발에만 최소 하루 소요 예상  
  * 오늘이 프로젝트 마감일  
* **주요 내용**

  * 사용자 테스트 실행 시간 부족  
  * 프로토타입 완성 시점에 따라 테스트 포함 여부 결정 필요  
  * 강의(관련 학습)는 프로토타입 완성 후 가능  
* **결정 사항**

  * **사용자 테스트/AB테스트는 Phase 2(다음 프로젝트)로 연기**  
  * 프로토타입 완성 시점에 따라 추후 협의  
  * 현재는 프로토타입 개발과 발표 준비에 모든 역량 집중
